Privacy Policy

1. Introduction

This Privacy Policy outlines how CRESSIDA (a static prototype website) handles information. Since this is a non-functional prototype for educational purposes, the scope of data collection and processing is extremely limited and simulated.

2. Information We "Collect" (Simulated)

For the purpose of simulating user registration and login, this static website may store the following information locally in your browser using cookies or browser local storage:

  • A simulated username.
  • A flag indicating a simulated "logged-in" state.

No personally identifiable information (PII) is collected, transmitted to any server, or stored persistently beyond your local browser session in a way that a real application would. All user data (usernames, passwords, balances) is hardcoded in JavaScript files for demonstration.

3. How We Use Your Information (Simulated)

The locally stored simulated information is used solely to demonstrate:

  • User registration and login functionality.
  • Personalized display elements (e.g., showing a username).
  • Simulated user-specific interactions (e.g., fake trades affecting a fake balance).

4. Data Security (Conceptual)

While this prototype does not handle real sensitive data, in a real-world application, appropriate security measures (encryption, secure servers, password hashing) would be implemented to protect user data. This prototype does not implement such measures as it is entirely client-side and uses hardcoded data.

5. Cookies

We use cookies purely to simulate a logged-in session. These cookies are stored locally in your browser and can be cleared through your browser settings. They do not track you across other websites.

6. Third-Party Services

This website uses:

  • Font Awesome for icons (via CDN).
  • Chart.js for charts (via CDN).

Please refer to their respective privacy policies if you have concerns about data handled by these third-party CDN providers.

7. Changes to This Policy

This placeholder privacy policy may be updated if the project scope changes. For now, it reflects the nature of a static, non-data-collecting prototype.
